# Customer Segmentation using Online Retail II Dataset

## Project Overview

Customer segmentation is a business intelligence technique used to group customers based on similar purchasing behaviors. It enables organizations to design targeted marketing campaigns, improve customer retention, optimize promotional strategies, and enhance customer experience.

This project performs customer segmentation using the **Online Retail II dataset** by applying **data preprocessing, feature engineering, dimensionality reduction (PCA), and K-Means clustering**. The resulting customer segments are analyzed to generate actionable business insights and marketing recommendations.

---

## Dataset

**Dataset:** Online Retail II

The dataset contains transactional records from an online retail store, including customer purchases, product information, invoice details, quantities, prices, and purchase timestamps.

### Key Features

- Invoice
- StockCode
- Description
- Quantity
- InvoiceDate
- Price
- Customer ID
- Country

## Project Workflow

### 1. Data Cleaning

The raw dataset was cleaned to improve data quality before analysis.

Performed operations:

- Removed missing Customer IDs
- Removed missing product descriptions
- Removed duplicate records
- Removed cancelled transactions
- Removed records with negative quantity
- Removed records with zero or negative price
- Converted InvoiceDate to datetime format
- Created Revenue feature

---

### 2. Exploratory Data Analysis (EDA)

Performed extensive exploratory analysis to understand customer purchasing patterns.

Visualizations include:

- Monthly Revenue Trend
- Daily Revenue Trend
- Revenue Distribution
- Top Revenue Generating Countries
- Top Selling Products
- Quantity Distribution
- Price Distribution
- Customer Purchase Frequency
- Revenue per Customer
- Correlation Heatmap

### 3. Feature Engineering

Customer-level features were generated by aggregating transactional records.

#### RFM Features

- Recency
- Frequency
- Monetary

#### Purchase Behavior Features

- Total Quantity Purchased
- Average Quantity

- Minimum Quantity

#### Pricing Features

- Average Price
- Maximum Price

#### Basket Features

- Average Basket Size

#### Product Features

- Product Diversity

#### Customer Activity Features

- Active Shopping Months
- Customer Lifetime

#### Time-based Features

- Weekend Purchase Ratio
- Morning Purchase Ratio
- Afternoon Purchase Ratio
- Evening Purchase Ratio

#### Geographic Feature

- Country (Encoded)

---

### 4. Feature Selection

To reduce redundancy and multicollinearity, highly correlated engineered features were removed before model training.

Removed features:

- CustomerValue
- RevenuePerProduct
- FrequencyPerMonth

---

### 5. Feature Scaling

All numerical features were standardized using:

- StandardScaler

This ensures equal contribution of each feature during PCA and clustering.

---

### 6. Principal Component Analysis (PCA)

Principal Component Analysis (PCA) was applied to reduce dimensionality while preserving the majority of information.

Steps performed:

- Standardized feature matrix
- Calculated explained variance ratio
- Generated cumulative explained variance plot
- Selected optimal principal components
- Generated 2D PCA visualization for cluster representation
- Analyzed PCA feature loadings

---

### 7. Customer Segmentation using K-Means

K-Means clustering was used to identify groups of customers with similar purchasing behavior.

The optimal number of clusters was determined using:

- Elbow Method
- Silhouette Score

Although the Silhouette Score suggested two clusters, four clusters were selected based on the Elbow Method to provide more meaningful and actionable customer segments for business analysis.

---

### 8. Customer Persona Generation

Customer personas were generated automatically by analyzing cluster statistics instead of manually assigning labels.

Each cluster was profiled using:

- Average Recency
- Average Frequency
- Average Monetary Value
- Average Basket Size
- Average Order Value
- Product Diversity

Generated customer personas include:

- Premium Loyal Customers
- Frequent Buyers
- Regular Customers
- Dormant Customers
- Low Value Customers
